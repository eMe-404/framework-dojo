container in dependency injection

in application context the runtime environment is called container, it will be instantiated during application start time.
after application instantiated you can retrieve the active `container` detail by `DojoContextUtils.getDojoContainer()`
you can see container initialized successfully message if it started successfully.
A Java class is a managed bean if it meets all the following conditions:
  • It is not an inner class.
  • It has an appropriate constructor - either:
    • the class has a constructor with no parameters
    • the class declares a constructor annotated `@Inject`
  • the class annotated with `@DojoComponent`
following two class both are managed bean. 
    
        @DojoComponent
        class Bar {
        }

        @DojoComponent
        class PaymentProcessorImpl implements PaymentProcessor {
        }
All `managed bean` will manged by `container` and loaded at application start time.
given following two bean 
        
        @DojoComponent
        class Baz {
        }

        @DojoComponent
        class Foo {
        }
Foo are able to retrieved by `container.retrieveBean` method: Foo
Baz are able to retrieved by `container.retrieveBean` method: Baz
